Modificadores de Acesso

Definem o escopo/visibilidade de um método/atributo

  • Permitem que sejam criadas as interfaces* entre as classes e o seu meio externo
  • Com eles, é possível determinar exatamente o que o meio externo poderá visualizar

Modificadores de Acesso

Em Java existem basicamente 4 modificadores de acesso diferentes

  • public (+): atributos/métodos podem ser acessados por objetos de outras classes
  • protected (#): membros da classe são acessíveis por objetos da mesma classe, objetos de classes filhas ou objetos de classes que estejam no mesmo pacote

Modificadores de Acesso

  • private (-): protege o método ou atributo do acesso externo, permitindo que sejam acessados da própria classe ou por objetos desta classe
  • default (package-private): quando modificadores de acesso não são especificados. Métodos/atributos podem ser acessados pela própria classe e por classes do mesmo pacote.

Modificadores de Acesso

Níveis de Acesso
Modificador Classe Pacote Subclasse Mundo
public S S S S
protected S S S N
no modifier S S N N
private S N N N

Modificadores de Acesso (Classes)

  • public: a classe é visível para todas as classes
  • default (package-private): quando um modificador não é especificado a classe é visíve somente para outras classes do mesmo pacote

Encapsulamento de Dados

Esconde o estado interno dos objetos (atributos) e requer que todas as operações sejam realizadas através dos métodos dos objetos
  • Detalhes de implementação ficam escondidos do mundo externo
  • Objeto mantém controle de como os atributos podem ser modificados
    • Evitar valores inválidos para um atributo

Getters and Setters

Um método get é um método sem parâmetros e com tipo de retorno definido, que retorna o valo de um determinado atributo.

Um método set é um método sem retorno (void) e com parâmetro(s), que altera o valor de um ou mais atributos.

Exemplo (getters e setters)


class Pessoa {
    private int idade;

    Pessoa(){
    }

    public void setIdade(int i){
        idade = i;
    }

    public int getIdade(){
        return idade;
    }
}

Exemplo (sem encapsulamento)


class Pessoa {
    public int idade;

    Pessoa(){
    }
}
					

//main em outra classe
public static void main(String args[]){
    Pessoa p = new Pessoa();
    p.idade = -1;
}

Exemplo (com encapsulamento)


class Pessoa {
    private int idade;

    Pessoa(){
    }

    public void setIdade(int i){
        if (i > 0) {
            idade = i;
        } else {
            System.out.println("Idade inválida");
        }
    }
}
					

//main em outra classe
public static void main(String args[]){
    Pessoa p = new Pessoa();
    p.setIdade(-1);
}

Exercício 1

Escreva uma classe que represente uma conta bancária

  • A conta armazena o nome, cpf e saldo do cliente
  • Deve ser possível operações de saque, depósito e visualização de saldo
  • Não deve ser permitido o saque em contas sem saldo
  • Os atributos devem ser encapsulados

Exercício 2

Escreva uma classe que represente um ponto no plano cartesiano

  • Coordenadas X e Y
  • Métodos para inicializar e calcular a distância até outro ponto
  • Os atributos devem ser encapsulados

Exercício 3 (entregar)

Escreva uma classe que represente um Círculo

  • Atributos: nome, raio e centro (objeto da classe Ponto)
  • Métodos
    • Inicializar os dados
    • Calcular diâmetro
    • Calcular área
    • Calcular circunferência
    • Modificar nome
    • Exibir os dados
  • Os atributos devem ser encapsulados

Exercício 4 (entregar)
